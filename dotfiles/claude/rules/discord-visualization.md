# discord-visualization

## Discord 시각화 정책

수치 데이터로 답할 때는 **텍스트 응답 전에** Discord 시각화 카드를 먼저 전송한다.

## 트리거 조건 (해당 시 필수)

- 디스크 사용률, 메모리, CPU 등 시스템 리소스 수치
- RAG 청크 수, DB 크기, 인덱싱 경과 시간
- 에러/경고 건수, 프로세스 수, 크론 상태
- "사용량", "얼마나", "몇 개", "상태 알려줘" 류 질문에 대한 숫자 응답

## 호출 방법

```bash
cd ~ && node ~/.jarvis/scripts/discord-visual.mjs \
  --type stats \
  --data '{"title":"제목","data":{"항목1":"값1","항목2":"값2"},"timestamp":"날짜"}' \
  --channel jarvis-system
```

## --type 선택 규칙

| 타입 | 용도 |
|------|------|
| `stats` | 범용 키-값 카드 (기본값) |
| `disk` | 디스크 전용 |
| `rag-health` | RAG 상태 전용 |
| `system-doctor` | 시스템 종합 진단 전용 |

전용 타입이 있으면 `stats` 대신 전용 타입 우선 사용.

## 예외 (전송 불필요)

- 코드 리뷰, 개념 설명 등 숫자 없는 순수 텍스트 답변

## 실패 처리

실패해도 exit 0 — 응답 차단 없음. 항상 시도할 것.

### AI Slop 방지 — 절대 사용 금지

AI가 만드는 진부한 패턴은 결과물의 신뢰도를 낮추고 "AI가 만들었네" 인상을 줍니다. 다음을 회피하십시오.

**금지 폰트**: Inter, Roboto, Arial, 시스템 기본 sans-serif
**금지 색상**: 흰 배경 위 보라 그라데이션, 푸른색 그라데이션
**금지 레이아웃**: 좌우 50:50 분할 카드 반복, 12-column grid 무비판 사용
**금지 컴포넌트 패턴**: 둥근 모서리 8px + 그림자 + 호버 lift 조합 일색

### 권장 디자인 베이스 (Opus 4.7 기본값)

다른 지침 없으면 다음을 채택하십시오 (편집·포트폴리오·일반 보고서에 잘 맞음):
- **배경**: 크림/오프화이트 (`#F4F1EA`)
- **서체**: 세리프 (Georgia, Fraunces, Playfair) — 헤딩 정각 산세리프, 본문 세리프
- **악센트**: 테라코타/앰버 (`#D4622A`)
- **레이아웃**: 중앙 정렬, 4px 둥근 모서리, 넓은 자간

### 재정의가 필요한 영역

다음 제품군은 위 기본값이 어색하므로 별도 팔레트 명세 필수:
- **대시보드·핀테크·헬스케어·엔터프라이즈**: 따뜻한 톤·세리프 폰트 사용 금지
- **개발툴·데이터 앱**: 크림 배경 부적합 — 다크 또는 차분한 회색 권장

### 자비스 컴포넌트별 카테고리 매핑 (충돌 방지)

| 컴포넌트 | 카테고리 | 권장 베이스 |
|---|---|---|
| 자비스맵 (`jarvis-board`) | 개발툴 | 다크 또는 차분한 회색 (현재: slate/indigo 유지 OK) |
| 이력서 (`/my-resume`) | 편집·포트폴리오 | 크림/세리프/테라코타 (Opus 4.7 기본값 적용) |
| 블로그 (`/blog-writer`) | 편집·포트폴리오 | 크림/세리프/테라코타 |
| 디스코드 시각화 카드 (`discord-visual.mjs`) | 데이터 앱 | 다크 또는 차분한 회색 |
| HTML 리포트 (브리핑·감사) | 편집·포트폴리오 | 크림/세리프/테라코타 |

**원칙**: 위 매핑이 없는 새 컴포넌트는 작업 시작 전 카테고리를 결정하고 그에 맞는 베이스를 사용하십시오. 매핑이 모호하면 룰 갱신을 먼저 하십시오.

### 명세 방법 2가지

**방법 1 — 구체 팔레트 명세** (정밀도 우선):
```
색상 팔레트: #E9ECEC, #C9D2D4, #44545B
서체: 헤딩 정각 산세리프, 넓은 자간
레이아웃: 중앙 정렬, 4px 둥근 모서리
따뜻한 톤 및 세리프 폰트 사용 금지
```

**방법 2 — 빌드 전 옵션 제안** (다양성 우선):
```
구현 전 이 브리프에 맞는 4가지 비주얼 방향을 제안해 주세요.
각각 배경색 / 악센트 / 서체 — 한 줄 설명 형식.
선택 후 그 방향만 구현하세요.
```
