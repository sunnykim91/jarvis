## 1. Purpose

`~/jarvis` is the **infra-heavy half** of the Jarvis ecosystem. It contains:

- 82 scheduled tasks (crons + event-triggered)
- A 24/7 Discord bot
- A RAG knowledge base (LanceDB + Ollama hybrid)
- `ask-claude.sh` — the LLM gateway used by every cron
- A virtual organization of ~11 AI teams that own cron subsets and Discord channels

Sister project: `~/jarvis-board` — a Next.js app that visualizes this ecosystem as a pixel-art virtual office ("자비스맵"). It is **read-only** from this repo; do not modify it here.

## 3. Key subsystems

### Cron scheduler

- **SSoT**: `~/.jarvis/config/tasks.json` (82 tasks)
- **Runner**: `infra/lib/task-store.mjs` + shell wrappers under `infra/scripts/`
- **Gateway**: Every LLM task goes through `infra/lib/llm-gateway.sh` → `ask-claude.sh`
- **Budgets**: `maxBudget` per task + `team-budget.json` per team cap + daily global cap
- **Reliability**: circuit breaker, retry with backoff, ledger (append-only JSONL), Tier 1-5 auto-tune

Full index: [TASKS-INDEX.md](TASKS-INDEX.md) (auto-generated by `gen-tasks-index.mjs`).

### Discord bot

- **Entry**: `infra/discord/discord-bot.js`
- **Runner**: `lib/claude-runner.js` → @anthropic-ai/claude-agent-sdk (1M context beta always on)
- **Process mgmt**: `ai.jarvis.discord-bot` LaunchAgent (long-running)
- **Handlers**: `infra/discord/lib/handlers.js` + `pre-processor.js`
- **PDF**: pdftotext → fallback to Claude Read
- **Streaming**: 1,900-char chunks to Discord
- **Full flow**: [ARCHITECTURE.md](ARCHITECTURE.md#message-flow)

### RAG knowledge base

- **Code**: `rag/` (indexer, query, schema)
- **Store**: `~/.jarvis/data/lancedb/` (LanceDB vector)
- **Embedding**: Ollama local (bge-m3 by default)
- **Query**: Hybrid BM25 + Dense + optional reranker
- **Live indexer**: `ai.jarvis.rag-watcher` LaunchAgent
- **Weekly compact**: `ai.jarvis.rag-compact` (Sun 03:00) — **must unload watcher first**
- **Destruction rule**: Never compact while watcher is loaded. See `~/.claude/rules/rag-system.md`

### LLM gateway (`ask-claude.sh`)

- All cron-driven LLM calls flow through this one script
- Uses `--mcp-config empty-mcp.json` (zero MCP) — only Claude Code CLI gets full MCP
- Reads `ANTHROPIC_API_KEY` or `CLAUDE_CODE_OAUTH_TOKEN`
- Emits token ledger rows to `~/.jarvis/logs/token-ledger.jsonl` for `token-ledger-audit`

### Virtual teams

11 AI team personas (`infra-lead`, `finance`, `trend-lead`, `record-lead`, `library`, `growth-lead`, `brand-lead`, `audit-lead`, `secretary`, `president`, `standup`). Defined in `~/jarvis-board/lib/map/team-registry.ts` (SSoT). Each team owns a subset of crons (see [TEAMS-CRONS.md](TEAMS-CRONS.md)) and posts to a specific Discord channel.

## 5. Entry points for AI agents

**Read in this order** (total < 2 minutes):

1. **`~/jarvis/CLAUDE.md`** — project-level policy (privacy, naming, emoji output rule)
2. **`~/jarvis/infra/docs/MAP.md`** — this file
3. **`~/jarvis/infra/docs/TASKS-INDEX.md`** — what exists, grouped by team
4. **Target-specific doc** — whichever of the following is closest to the task:
   - `TEAMS-CRONS.md` if user mentions a team
   - `CONFIG.md` if user is changing config
   - `ARCHITECTURE.md` if user is debugging the Discord bot or LLM runner
   - `OPERATIONS.md` if user is firefighting
   - `SYSTEM-OVERVIEW.md` for current runtime snapshot

Then read the specific script with Serena (`find_symbol`, `get_symbols_overview`) — **not** full-file `Read` for scripts > 500 lines.

## 7. Don'ts

- Don't hand-edit `effective-tasks.json` (runtime cache; `cron-sync.sh` regenerates it)
- Don't compact RAG while `rag-watcher` is loaded — 2 prior DB destructions
- Don't hardcode user paths; use `BOT_HOME` / `JARVIS_HOME` / `JARVIS_RAG_HOME`
- Don't put prompts in `ask-claude.sh` with MCP enabled — gateway always passes `empty-mcp.json`
- Don't Read 1,500+ line files without trying Serena first (`find_symbol` / `get_symbols_overview`)
- Don't commit `.env`, `jira.env`, `tutoring-platform.env`, `google-calendar-token.json`, or anything under `~/.jarvis/config/secrets/`
- Don't add a new team by creating a keyword list here — edit `~/jarvis-board/lib/map/team-registry.ts` first
