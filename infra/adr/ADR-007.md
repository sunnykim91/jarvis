# ADR-007: File-Convention Plugin System

- 상태: accepted
- 날짜: 2026-03-06
- 관련: ADR-005 (무상태 크론 실행), ADR-006 (LLM Gateway)

## 맥락

tasks.json에 29개 태스크가 하드코딩. 새 태스크 추가 시 JSON 직접 편집 필요.
오픈소스 시 커뮤니티가 태스크를 공유/설치할 수 있는 표준 구조 필요.

## 결정

`~/.jarvis/plugins/` 디렉토리에 파일 컨벤션 기반 플러그인 시스템 도입.

### 플러그인 구조
```
plugins/
  my-task/
    manifest.json   # 필수: tasks.json 엔트리 호환 형식
    context.md      # 선택: 시스템 프롬프트
    test.sh         # 선택: 자가 테스트
    README.md       # 선택: 문서
```

### 동작 방식
- `plugin-loader.sh`가 plugins/ 스캔 → tasks.json과 병합 → effective-tasks.json 생성
- `jarvis-cron.sh`는 effective-tasks.json 우선 사용, 없으면 tasks.json 폴백
- 개인 설정(tasks.json)과 공유 가능 플러그인(plugins/) 분리
- `.` 접두사 디렉토리는 예제/템플릿으로 자동 제외

## 대안

- npm 패키지 방식: 과도한 복잡성. 단순 디렉토리 컨벤션이 적절.
- tasks.json 자체를 generated로: 수동 편집 불가능해짐.

## 결과

- 커뮤니티 태스크 공유: `git clone URL ~/.jarvis/plugins/name`
- Lite/Company 모드 전환 시 플러그인 활성화/비활성화로 대응
- 기존 tasks.json 호환성 100% 유지
