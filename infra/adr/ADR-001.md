# ADR-001: claude -p 기반 아키텍처

- **상태**: accepted
- **날짜**: 2026-02-28
- **결정자**: Owner + Jarvis

## 맥락 (Context)
Claude Max $330/월을 이미 지불하고 있었다. the previous project(종량제 API 기반)에 월 $36 추가 비용이 발생하고 있었고, 모델 품질도 Gemini Flash/GPT-5-mini 수준에 머물렀다. 동일 비용으로 Opus 급 모델을 활용할 방법이 필요했다.

## 결정 (Decision)
`claude -p` (headless CLI)를 래핑하는 `ask-claude.sh`를 단일 실행 엔진으로 채택했다. 모든 크론 태스크, Discord 봇 응답, 팀별 에이전트가 이 래퍼를 통해 실행된다. 핵심 인터페이스: `ask-claude.sh TASK_ID PROMPT [ALLOWED_TOOLS] [TIMEOUT] [MAX_BUDGET]`.

## 대안 (Alternatives)
- OpenAI API(기존 the previous project 구성): gpt-4.1 $2/$8 단가로 크론 48개 운영 시 월 $18~36 — 선택하지 않은 이유: 이미 지불 중인 Claude Max를 활용하지 않는 비효율. 모델 품질도 Opus 대비 열위.
- Gemini API(OpenRouter 경유): 저렴하나 복잡한 분석 태스크에서 품질 부족. 긴 컨텍스트 처리에서 hallucination 빈도 높음.
- 자체 호스팅 LLM(Ollama llama3.2 등): Mac Mini에서 3B 모델만 실용적. 크론 태스크에 필요한 분석 품질에 턱없이 부족.

## 결과 (Consequences)
- 긍정: 월 $36 절약($432/년). 모델 품질 Gemini Flash에서 Opus 4.6으로 대폭 상승. 인프라 단순화(API 키 관리, 게이트웨이 서버 불필요).
- 부정: Rate limit 리스크(일 150msg/5hr 900 기준 17% 사용). ToS 회색지대(자동화 사용). claude CLI 업데이트 시 breaking change 가능.
- 중립: the previous project 게이트웨이(포트 18789) 비활성화. the previous project LaunchAgents는 *.plist.disabled로 보존.
