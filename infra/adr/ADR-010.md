# ADR-010: MCP Resources — URI 기반 읽기 전용 상태 접근

- 상태: accepted
- 날짜: 2026-03-10
- 관련: ADR-009 (Nexus 3분리)

## 맥락

MCP Tools는 동작(실행, 검색, 상태조회)에 적합하지만, 정적 상태값(현재 캐시 크기,
시스템 헬스 요약, 최근 로그)을 요청할 때마다 Tool 호출을 통해 실행 비용을
치르는 것은 비효율적임.

MCP 사양은 Resources 개념을 통해 URI로 식별되는 읽기 전용 데이터를
구조화된 방식으로 노출하는 표준 패턴을 제공함.

## 결정

Nexus MCP 서버에 3개의 리소스 엔드포인트를 추가:

| URI | 설명 | MIME |
|-----|------|------|
| `jarvis://health` | 최신 health.json 요약 | application/json |
| `jarvis://logs` | cron.log 최근 50줄 | text/plain |
| `jarvis://cache/stats` | exec 캐시 크기·히트율 | application/json |

### 구현

```javascript
server.setRequestHandler(ListResourcesRequestSchema, async () => ({
  resources: [
    { uri: 'jarvis://health',      name: '시스템 헬스', mimeType: 'application/json' },
    { uri: 'jarvis://logs',        name: '크론 로그',   mimeType: 'text/plain'        },
    { uri: 'jarvis://cache/stats', name: '캐시 통계',   mimeType: 'application/json' },
  ],
}));

server.setRequestHandler(ReadResourceRequestSchema, async (req) => {
  const uri = req.params.uri;
  // ... switch on uri, read file or call getCacheStats()
  return { contents: [{ uri, mimeType, text }] };
});
```

### capabilities 선언

```javascript
new Server({ name: 'nexus-cig', version: '3.0.0' },
           { capabilities: { tools: {}, resources: {} } });
```

## 대안

- Tool만 사용: `health` 도구를 매번 실행. 부작용 없어도 Tool로 처리하는 개념 혼선.
- 파일 직접 노출: MCP Resources는 URI 추상화를 제공하므로 파일 경로 변경에 무관.
- SSE/Streaming: 실시간 스트림이 필요한 경우에 적합하나, 현재 요구사항 초과.

## 결과

- 상태 조회와 동작 실행의 개념적 분리 (Resources vs Tools)
- Claude가 `jarvis://` URI를 통해 현재 시스템 상태를 낮은 비용으로 참조 가능
- tools 목록이 길어지지 않아 컨텍스트 절약
- MCP 표준 준수로 향후 MCP 클라이언트 생태계 호환성 확보
