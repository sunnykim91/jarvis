# ADR-011 — Task FSM + SQLite 단일 저장소

**상태:** accepted
**날짜:** 2026-03-17
**결정자:** Owner

---

## Context

Jarvis의 자율 개발 큐(`dev-queue.json`)는 Python3 인라인 스크립트로 JSON 파일을 직접 읽고 덮어썼다. 문제:

1. **원자성 없음** — `json.dump()` 도중 프로세스 죽으면 파일 손상
2. **상태 전이 규칙 없음** — `done → running` 같은 무효 전이가 코드 실수로 가능
3. **히스토리 없음** — `running` 상태로 5시간 방치돼도 감지 불가
4. **`skipped` 복구 불가** — JSON 수동 편집 외 방법 없음

태스크 상태 관리 라이브러리 도입을 검토했다.

---

## 검토한 대안

| 옵션 | 거절 이유 |
|------|-----------|
| **LangGraph** | LLM이 "다음 노드를 직접 결정"하는 멀티 에이전트 그래프용 도구. 태스크 FSM은 LLM 분기 문제가 아님. Python 런타임 의존성, 수십 MB 설치 부담 |
| **XState** | `@xstate/fsm`이 v5에서 공식 폐기. 상태 6개·전이 10개 미만 FSM에 외부 라이브러리 유지보수 비용 부담 |
| **better-sqlite3** | native addon (node-gyp 필요). Node.js 버전 업그레이드 시 매번 rebuild. `~/.jarvis/node_modules`에 미설치 |
| **JSON 유지 + SQLite 히스토리 추가** | 이중 저장소는 반드시 불일치 발생. 최악의 선택 |

---

## 결정

### 1. 직접 구현 FSM — 순수 함수 (`lib/task-fsm.mjs`)

부수효과 없는 순수 함수만 포함. 저장소 교체에 독립적.

```js
export function canTransition(from, to)  // guard
export function applyTransition(task, to) // 새 객체 반환, 무효 전이 시 throw
export function pickNextTask(tasks[])    // 의존성 충족 + priority 최고값 선택
```

### 2. node:sqlite 내장 모듈 (`lib/task-store.mjs`)

Node.js 22.5+에 내장. 별도 설치 불필요. WAL 모드 + `BEGIN/COMMIT/ROLLBACK` 직접 제어로 원자적 전이.
(`node:sqlite`의 `DatabaseSync`는 `better-sqlite3` 스타일의 `.transaction()` 헬퍼를 지원하지 않음)

```sql
tasks            (id, status, priority, retries, depends, meta, updated_at)
task_transitions (task_id, from_status, to_status, triggered_by, created_at)
```

`transition()` 호출 시 `UPDATE tasks` + `INSERT task_transitions`가 `BEGIN/COMMIT` 블록으로 묶임. 실패 시 `ROLLBACK` 후 에러 재발생.

### 3. dev-queue.json → tasks.db 완전 이전

Python3 인라인 7곳 + jq 3곳 → `node task-store.mjs` CLI 호출로 교체.
insight-extractor.mjs, extras-gateway.mjs도 동시 전환.

---

## 결과

- `pending / queued / running / done / failed / skipped` 6개 상태, 허용 전이 외 차단
- 모든 상태 변경이 `task_transitions` 테이블에 timestamp + triggered_by와 함께 기록
- `skipped → pending` 복구: `node task-store.mjs transition <id> pending manual-fix`
- `INSERT OR IGNORE`로 중복 태스크 등록 방지
- 외부 의존성 추가 0건
