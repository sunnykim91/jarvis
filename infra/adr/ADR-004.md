# ADR-004: Obsidian Vault 지식 베이스

- **상태**: accepted
- **날짜**: 2026-03-05
- **결정자**: Owner + Jarvis

## 맥락 (Context)
에이전트 산출물(트렌드 리포트, KPI, 학습 자료 등)이 results/ 디렉토리에 7일 후 삭제되고 있었다. 사용자가 Galaxy/MacBook에서 탐색하고, 에이전트가 장기 지식으로 재활용할 수 있는 영속 저장소가 필요했다.

## 결정 (Decision)
로컬 Obsidian Vault(`$VAULT_DIR/`)를 지식 베이스로 사용한다. LanceDB 하이브리드 검색(Vector + BM25)으로 시맨틱 검색을 지원하고, ask-claude.sh의 RAG 파이프라인(`rag-query.mjs`)이 실행 시 자동으로 관련 지식을 주입한다. 매시간 크론으로 증분 인덱싱. 에이전트 산출물 중 인사이트는 `record_insight()` 함수가 자동으로 Vault에 저장.

## 대안 (Alternatives)
- Notion API: 클라우드 기반으로 Galaxy 접근 용이 — 선택하지 않은 이유: API rate limit, 오프라인 불가, 마크다운 호환성 제한. 에이전트가 직접 파일 I/O 불가.
- GitHub Wiki: 버전 관리 내장 — 선택하지 않은 이유: 검색 기능 빈약. 로컬 파일 접근 불가로 에이전트 RAG 파이프라인 연동 불가능.
- 단순 파일 시스템(Obsidian 없이): 추가 도구 불필요 — 선택하지 않은 이유: 인간의 탐색/연결 경험 부재. Dataview 쿼리, 그래프 뷰, 태그 시스템 등 지식 관리 기능 포기.

## 결과 (Consequences)
- 긍정: Galaxy iCloud 동기화로 모바일 접근 가능. Dataview 플러그인으로 대시보드 구성. frontmatter 태그로 체계적 분류. 에이전트 RAG와 인간 탐색이 동일 소스 사용(SSoT).
- 부정: Obsidian 앱 의존(유료 Sync 없이 iCloud 사용 시 충돌 가능). LanceDB 인덱싱에 OpenAI Embedding API 비용 발생(text-embedding-3-small, 미미하나 존재).
- 중립: Vault 구조는 `01-inbox/`~`06-knowledge/` 6단계 PARA 변형. 에이전트 산출물은 `02-daily/insights/`에 자동 축적.

## 관련 결정
- ADR-003: 파일 기반 에이전트 간 통신 (단기 통신 vs 장기 저장의 역할 분리)
- ADR-001: claude -p 기반 아키텍처 (RAG 컨텍스트를 system prompt에 주입하는 구조)
