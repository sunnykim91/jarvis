# ADR-009: Nexus MCP Gateway 3분리 (exec / rag / health)

- 상태: accepted
- 날짜: 2026-03-10
- 관련: ADR-006 (LLM Gateway), ADR-007 (플러그인 시스템)

## 맥락

`lib/mcp-nexus.mjs` 단일 파일이 1,000줄을 초과하여 유지보수성이 저하됨.
exec(명령 실행), rag(검색), health(상태확인) 세 도메인이 혼재하여 수정 시
전체 파일을 로드해야 하고, 도메인별 책임 경계가 불명확함.

또한 scan() 도구에서 무제한 병렬 프로세스 생성으로 macOS ulimit 도달 위험이 있음.

## 결정

단일 nexus 파일을 3+1 구조로 분리:

```
lib/
  mcp-nexus.mjs          ← 얇은 오케스트레이터 (라우팅만)
  nexus/
    shared.mjs            ← 공통 유틸 (runCmd, compress, logTelemetry, mkResult/mkError)
    exec-gateway.mjs      ← exec, scan, cache_exec, log_tail, file_peek
    rag-gateway.mjs       ← rag_search (LanceDB 하이브리드)
    health-gateway.mjs    ← health (LaunchAgent, 리소스, 크론 로그)
```

### 라우팅 메커니즘

```javascript
const GATEWAYS = [execGateway, ragGateway, healthGateway];
// 각 gateway.handle(name, args, start) → null(미처리) or result
for (const gw of GATEWAYS) {
  const result = await gw.handle(name, args, start);
  if (result !== null) return result;
}
```

첫 번째 non-null 결과가 응답됨. 새 게이트웨이 추가는 배열에 push만 하면 됨.

### 병렬 프로세스 제한

```javascript
const MAX_CONCURRENT_PROCS = 8;
async function mapLimited(items, limit, fn) {
  for (let i = 0; i < items.length; i += limit) {
    results.push(...await Promise.allSettled(items.slice(i, i + limit).map(fn)));
  }
  return results;
}
```

외부 의존성 없이 배치 처리. p-limit 패키지 불필요.

## 대안

- 단일 파일 유지 + 내부 함수 정리: 1,500줄 제한 위반 지속, 근본 해결 아님.
- 4개 독립 MCP 서버: 각각 stdio transport를 가져야 하므로 discord-mcp.json 수정 필요. 진입점 복잡도 증가.
- 동적 import + 플러그인 자동 감지: 불필요한 복잡도. 게이트웨이 수가 3개로 안정적.

## 결과

- 파일당 줄수: 300줄 이하 (기존 1,000줄 → 분산)
- 진입점 변경 없음: `node lib/mcp-nexus.mjs` 그대로
- discord-mcp.json 변경 없음
- scan() 최대 동시 프로세스 8개로 제한
- 게이트웨이 추가 시 GATEWAYS 배열 한 줄 추가로 확장
