# ADR-002: 4계층 토큰 격리

- **상태**: accepted
- **날짜**: 2026-02-28
- **결정자**: Owner + Jarvis

## 맥락 (Context)
`claude -p`는 기본적으로 작업 디렉토리의 CLAUDE.md, .claude/ 설정, MCP 서버 등을 모두 로드한다. 크론 태스크에서 이 컨텍스트가 불필요하게 토큰을 소비하며, 50K 토큰이 5K만으로 충분한 작업에도 낭비되고 있었다.

## 결정 (Decision)
4개 계층으로 토큰 유입을 차단한다.
1. `/tmp/bot-work/{task}-{pid}` 작업 디렉토리 + 가짜 `.git/HEAD` (부모 repo 탐색 차단)
2. `--strict-mcp-config --mcp-config empty-mcp.json` (MCP 서버 제로)
3. `--append-system-prompt`로 태스크별 최소 프롬프트만 주입 (context/{task}.md + results/ 최근 3건)
4. `--plugin-dir .empty-plugins` + `--setting-sources local` (외부 플러그인/설정 차단)

## 대안 (Alternatives)
- 프롬프트 최적화만: system prompt를 줄이되 환경 격리 없음 — 선택하지 않은 이유: CLAUDE.md, MCP 서버 자동 로딩을 막을 수 없어 근본 해결 불가.
- MCP 제한만(empty-mcp.json만 적용): 디렉토리 컨텍스트(.claude/, CLAUDE.md)는 여전히 로드됨. 토큰 절약 효과 제한적.
- Docker 컨테이너 격리: 완벽한 격리이나 Mac Mini에서 Docker 오버헤드 과다. 크론 180초 타임아웃 내 컨테이너 기동 부담.

## 결과 (Consequences)
- 긍정: 태스크당 토큰 50K에서 5K로 90% 절감. Rate limit 여유 확보. 크론 태스크 간 컨텍스트 오염 원천 차단.
- 부정: 크론 태스크에서 MCP 도구(Brave Search, GitHub 등) 사용 불가. 복잡한 태스크는 allowed tools로 Read/Bash만 제한적 사용.
- 중립: Claude Code 인터랙티브 세션(tmux claude1/claude2)은 이 격리 적용 대상 아님. 대화형 작업은 풀 컨텍스트 유지.

## 관련 결정
