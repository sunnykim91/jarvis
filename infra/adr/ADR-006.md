# ADR-006: LLM Gateway Multi-Provider

- 상태: accepted
- 날짜: 2026-03-06
- 관련: ADR-001 (claude -p 기반 아키텍처)

## 맥락

Jarvis의 모든 AI 호출이 `claude -p` 단일 경로에 의존. Anthropic이 CLI 동작을 변경하거나 rate limit을 강화하면 전체 시스템 중단. 4팀 합동 분석에서 만장일치 1순위 리스크로 선정 (Impact 9/10).

## 결정

`~/.jarvis/lib/llm-gateway.sh`를 통한 multi-provider fallback chain 도입.

### Provider Chain
1. **claude -p** (primary) — Claude Max 구독, $0, 도구 사용 가능
2. **Anthropic API** (fallback) — ANTHROPIC_API_KEY 설정 시, 텍스트 전용
3. **OpenAI API** (fallback) — OPENAI_API_KEY 설정 시 (GPT-4o), 텍스트 전용
4. **Ollama** (fallback) — 로컬 llama3.2, 텍스트 전용

### 핵심 제약
- 도구 사용(Bash, Write 등) 필요 태스크는 claude -p만 가능. 폴백 불가.
- 폴백 프로바이더의 출력은 claude -p JSON 형식으로 변환하여 호환성 유지.

### 호출 인터페이스
```bash
source "$BOT_HOME/lib/llm-gateway.sh"
llm_call --prompt "..." --system "..." --timeout 180 \
         --model "..." --output "/tmp/out.json" \
         [--allowed-tools "Read,Bash"] [--max-budget "1.00"]
```

## 대안

- Anthropic API 전환: 월 $50+ 비용 발생. ADR-001 결정과 상충.
- Ollama 전용: 품질 불충분 (llama3.2:3b).

## 결과

- claude -p 장애 시 텍스트 전용 태스크(system-health, stock-monitor 등)는 자동 폴백
- 도구 필요 태스크(council-insight, record-daily 등)는 claude -p 필수 유지
- ask-claude.sh가 llm-gateway.sh를 source하여 기존 로직 최소 변경
